# TradeNexus — API Contracts

All request/response shapes across every service boundary. This is the single source of truth for data contracts. Update this file whenever a shape changes **before** touching implementation.

> **Business Logic Vigilance:** Any shape that touches money, prices, or orders must be verified against `docs/BUSINESS_LOGIC.md`. The wrong fee formula or P&L calculation in a contract will propagate through every layer.

---

## Canonical Data Types

### Instrument
```typescript
{
  instrumentId: string          // Primary key, e.g., "GOOGL", "US10Y"
  ticker: string                // Yahoo Finance symbol or FRED series ID
  categoryId: "STOCK" | "GOVT" | "ETF"
  instrumentDescription: string
  maxQuantity: number
  minQuantity: number
  couponRate?: number           // Only for GOVT: annual coupon rate (e.g., 0.0425 = 4.25%)
  maturityDate?: string         // Only for GOVT: ISO date string "YYYY-MM-DD"
}
```

### Price
```typescript
{
  instrumentId: string
  ticker: string
  askPrice: number              // lastPrice + halfSpread
  bidPrice: number              // lastPrice - halfSpread
  lastPrice: number             // Yahoo regularMarketPrice or bond PV
  priceTimestamp: string        // ISO 8601 — reject if > 60s old at trade time
  marketOpen: boolean           // NYSE open Mon-Fri 09:30-16:00 ET
  instrument: Instrument        // Embedded instrument details
}
```

### Client
```typescript
{
  email: string
  clientId: string              // UUID, generated by Spring Boot on register
  name: string
  dateOfBirth: string           // "YYYY-MM-DD"
  country: "India" | "USA"
  identification: ClientIdentification[]
  isAdmin: boolean
}
```

### ClientIdentification
```typescript
{
  type: "Aadhar" | "PAN" | "SSN"
  value: string
}
```

### ClientProfile (returned on login/register)
```typescript
{
  client: Client
  token: string                 // JWT string signed by Spring Boot (HS256, 30-min expiry)
}
```

### ClientPortfolio
```typescript
{
  clientId: string
  currBalance: number           // Current cash balance (NUMERIC 19,4)
  holdings: Holding[]
}
```

### Holding
```typescript
{
  instrumentId: string
  instrumentDescription: string  // Joined from INSTRUMENT table
  categoryId: string             // Joined from INSTRUMENT table
  quantity: number
  avgPrice: number               // Weighted average cost basis (includes BUY fee)
}
```

### ClientPreferences
```typescript
{
  clientId: string
  investmentPurpose: "Education" | "Major Expense" | "Retirement"
  incomeCategory: "LIG" | "MIG" | "HIG" | "VHIG"
  lengthOfInvestment: "Short" | "Medium" | "Long"
  percentageOfSpend: "Tier1" | "Tier2" | "Tier3" | "Tier4"
  riskTolerance: number         // 1–5 (1 = conservative, 5 = aggressive)
  acceptAdvisor: boolean        // was string 'true'/'false' — now boolean
}
```

### Order
```typescript
{
  orderId: string               // UUID v4, generated by frontend
  instrumentId: string
  quantity: number
  targetPrice: number           // For LIMIT: max willing to pay (BUY) or min to accept (SELL)
  direction: "B" | "S"
  orderType: "MARKET" | "LIMIT" // NEW: replaces 5% price tolerance
  clientId: string
  token: string                 // JWT — also sent as Authorization header (body is audit trail)
}
```

### Trade
```typescript
{
  tradeId: string               // UUID v4, generated by Spring Boot
  order: Order
  instrumentId: string          // Denormalized from order
  quantity: number              // Denormalized from order
  direction: "B" | "S"         // Denormalized from order
  clientId: string              // Denormalized from order
  executionPrice: number        // Actual price at execution (ask for BUY, bid for SELL)
  cashValue: number             // BUY: qty × askPrice × 1.001 | SELL: qty × bidPrice × 0.999
  executedAt: string            // ISO 8601 timestamp
}
```

**Fee note:** 0.1% applied on both sides. BUY cashValue is total cash debited (includes fee). SELL cashValue is net proceeds received (after fee deduction). Average cost basis for holdings includes the BUY fee.

### TradeHistory
```typescript
{
  clientId: string
  trades: Trade[]
}
```

### TradePL
```typescript
{
  instrumentId: string
  instrumentDescription: string
  categoryId: string
  realizedPL: number            // Sum of: (saleProceeds - avgCostBasis × qty) per sell
  unrealizedPL: number          // (currentBidPrice - avgPrice) × currentQuantity
  totalPL: number               // realizedPL + unrealizedPL
}
```

### MarketStatus
```typescript
{
  marketOpen: boolean
  timezone: string              // "America/New_York"
  currentTime: string           // ISO 8601
  nextOpenAt: string | null     // ISO 8601 — null if market is currently open
  nextCloseAt: string | null    // ISO 8601 — null if market is currently closed
}
```

### IsVerifiedClient
```typescript
{
  isVerified: boolean
}
```

### ErrorResponse (standardized across all services)
```typescript
{
  status: number                // HTTP status code
  message: string               // Error code from BUSINESS_LOGIC.md, e.g., "MARKET_CLOSED"
  timestamp: string             // ISO 8601
}
```

## WebSocket Message Types

### WS: Auth Frame (Browser → MDS, sent immediately after connection)
```typescript
{ type: "AUTH", token: string }
```

### WS: Auth Error (MDS → Browser)
```typescript
{ type: "AUTH_ERROR", message: "Invalid token" }
```

### WS: Price Snapshot (MDS → Browser, sent immediately after successful AUTH)
```typescript
{
  type: "PRICE_SNAPSHOT"
  prices: Price[]
  timestamp: string             // ISO 8601
}
```

### WS: Price Update (MDS → Browser, on every fetch cycle)
```typescript
{
  type: "PRICE_UPDATE"
  prices: Price[]
  timestamp: string
}
```

### WS: Heartbeat
```typescript
{ type: "PING" }   // MDS → Browser, every 30s
{ type: "PONG" }   // Browser → MDS, in response to PING
```

## MDS Endpoints (`http://localhost:3001`)

No authentication required on REST endpoints — prices are public market data. WebSocket requires AUTH frame.

### Instruments

**GET /instruments**
- Auth: None
- Response 200: `Instrument[]` (full 12-instrument list)

### Prices

**GET /prices**
- Auth: None
- Query: `?category=STOCK|GOVT|ETF` (optional filter)
- Response 200: `Price[]` (current in-memory cache snapshot)

**GET /prices/:instrumentId**
- Auth: None
- Response 200: `Price`
- Response 404: `ErrorResponse`

**GET /prices/:instrumentId/history**
- Auth: None
- Query: `?period=30d` (default; only 30d supported)
- Response 200: `Price[]` (rolling history — used by robo advisor momentum calc)
- Response 404: `ErrorResponse`

### Market Status

**GET /market-status**
- Auth: None
- Response 200: `MarketStatus`

### Health

**GET /health**
- Auth: None
- Response 200: `{ status: "ok", priceCount: 12, lastUpdate: "<ISO>" }`

### WebSocket

**WS ws://localhost:3001/ws/prices**
- Send `{ type: "AUTH", token: "<JWT>" }` immediately after connect
- Receive `PRICE_SNAPSHOT` on successful auth
- Receive `PRICE_UPDATE` on every fetch cycle (15s stocks, 60s bonds)
- Respond to `PING` with `PONG` or connection closes after 30s

## Spring Boot Endpoints (`http://localhost:8080`)

### Auth

**POST /auth/login**  ← was `GET /client?email=x&password=y` — security fix
- Body: `{ email: string, password: string }` (plaintext over HTTPS — backend BCrypts)
- Response 200: `ClientProfile`
- Response 400: `ErrorResponse` — `INVALID_CREDENTIALS`
- Response 404: `ErrorResponse` — `EMAIL_NOT_FOUND`

**POST /auth/register**
- Body: `{ email, password, name, dateOfBirth, country, identification[] }`
- Response 200: `ClientProfile`
- Response 400: `ErrorResponse` — `EMAIL_TAKEN` | `ID_TAKEN` | `INVALID_PASSWORD_FORMAT` | `INVALID_COUNTRY` | `MISSING_REQUIRED_FIELDS`

**POST /auth/refresh**
- Headers: `Authorization: Bearer <existing-token>`
- Response 200: `{ token: string, expiresAt: string }`
- Response 401: `ErrorResponse` — token expired or invalid

### Client

**GET /client/verify-email/:email**
- Auth: None (used during registration flow before login exists)
- Response 200: `IsVerifiedClient`

**GET /client/ping** → `"Client service is alive"`

### Instruments

**GET /instrument**
- Auth: Required (JWT)
- Response 200: `Instrument[]` (from DB, seeded by Flyway V2 migration)

### Trades

**GET /trade/ping** → `"Trade service is alive"`

**POST /trade/execute-trade**
- Auth: Required (JWT)
- Body: `Order` (include `orderType: "MARKET" | "LIMIT"`)
- Response 200: `Trade`
- Response 400: `ErrorResponse` — validation failure
- Response 402: `ErrorResponse` — `INSUFFICIENT_BALANCE`
- Response 409: `ErrorResponse` — `MARKET_CLOSED` | `LIMIT_NOT_MET` | `PRICE_DATA_STALE`

**GET /trade/live-prices**
- Auth: Required (JWT)
- Response 200: `Price[]` (REST fallback — frontend should prefer MDS WebSocket)

**POST /trade/suggest-buy**
- Auth: Required (JWT)
- Body: `ClientPreferences`
- Response 200: `Price[]` (up to 5, scored by 3-factor model)
- Response 403: `ErrorResponse` — `ADVISOR_DISABLED`

**POST /trade/suggest-sell**
- Auth: Required (JWT)
- Body: `{ clientId: string, preferences: ClientPreferences }`
- Response 200: `Holding[]` (up to 5, ranked by loss/concentration)
- Response 403: `ErrorResponse` — `ADVISOR_DISABLED`

### Portfolio

**GET /portfolio/ping** → `"Portfolio service is alive"`

**GET /portfolio/client/:clientId**
- Auth: Required (JWT; must own or isAdmin)
- Response 200: `ClientPortfolio`

### Client Preferences

**GET /client-preferences/ping** → `"Preferences service is alive"`

**GET /client-preferences/:clientId**
- Auth: Required (JWT)
- Response 200: `ClientPreferences`

**POST /client-preferences**
- Auth: Required (JWT)
- Body: `ClientPreferences`
- Response 200: `ClientPreferences`

**PUT /client-preferences**
- Auth: Required (JWT)
- Body: `ClientPreferences`
- Response 200: `ClientPreferences`

### Activity Report

**GET /activity-report/holdings/:clientId**
- Auth: Required (JWT; must own or isAdmin)
- Response 200: `Holding[]`

**GET /activity-report/trades/:clientId**
- Auth: Required (JWT; must own or isAdmin)
- Response 200: `TradeHistory`

**GET /activity-report/pl/:clientId**
- Auth: Required (JWT; must own or isAdmin)
- Response 200: `TradePL[]`
